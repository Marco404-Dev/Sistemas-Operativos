# Banco de preguntas — Gestión de procesos

## 1. ¿Qué es un proceso?
A) Un dispositivo de entrada  
B) Un programa en ejecución  
C) Un archivo comprimido  
D) Un tipo de memoria  

**Respuesta:** B) Un programa en ejecución

---

## 2. ¿Qué es un programa?
A) Un programa en ejecución  
B) Un archivo con instrucciones de forma estática  
C) Un recurso de hardware  
D) Un proceso terminado  

**Respuesta:** B) Un archivo con instrucciones de forma estática

---

## 3. ¿Cuál es la diferencia principal entre programa y proceso?
A) No hay diferencia  
B) El programa es dinámico y el proceso es estático  
C) El programa es estático y el proceso es dinámico  
D) Ambos son hardware  

**Respuesta:** C) El programa es estático y el proceso es dinámico

---

## 4. ¿Qué recurso se considera importante y limitado en la gestión de procesos?
A) El teclado  
B) La CPU  
C) El mouse  
D) La impresora  

**Respuesta:** B) La CPU

---

## 5. ¿Qué es la multiprogramación?
A) Tener varias computadoras conectadas  
B) Tener varios discos duros  
C) La CPU cambia rápidamente entre procesos dando la impresión de simultaneidad  
D) El uso de una sola aplicación  

**Respuesta:** C) La CPU cambia rápidamente entre procesos dando la impresión de simultaneidad

---

## 6. ¿Qué es un multiprocesador?
A) Un sistema con varios CPU ejecutando procesos al mismo tiempo  
B) Un sistema con una sola memoria RAM  
C) Un sistema sin sistema operativo  
D) Un sistema que solo ejecuta un programa  

**Respuesta:** A) Un sistema con varios CPU ejecutando procesos al mismo tiempo

---

## 7. ¿Qué elementos conforman básicamente un proceso?
A) Archivo y red  
B) Programa y contexto  
C) Mouse y teclado  
D) Usuario y contraseña  

**Respuesta:** B) Programa y contexto

---

## 8. ¿Qué hace el sistema operativo cuando se crea un proceso?
A) Apaga la computadora  
B) Borra el disco  
C) Reserva espacio para almacenar su información  
D) Elimina otros procesos automáticamente  

**Respuesta:** C) Reserva espacio para almacenar su información

---

## 9. ¿Cuál de los siguientes es un caso de creación de procesos?
A) Inicio del sistema  
B) Petición del usuario  
C) Llamada al sistema  
D) Todas las anteriores  

**Respuesta:** D) Todas las anteriores

---

## 10. ¿Qué significa PID?
A) Parent Identification Data  
B) Process ID  
C) Program Internal Data  
D) Process Input Device  

**Respuesta:** B) Process ID

---

## 11. ¿Qué significa PPID?
A) Process Program Internal Data  
B) Parent Process ID  
C) Primary Process Identification  
D) Program Process Input Device  

**Respuesta:** B) Parent Process ID

---

## 12. ¿Cómo se llama la estructura donde el sistema operativo guarda la información de cada proceso?
A) FAT  
B) BIOS  
C) PCB  
D) HTML  

**Respuesta:** C) PCB

---

## 13. ¿Qué significa PCB?
A) Process Control Block  
B) Primary Computer Base  
C) Program Count Block  
D) Process Central Board  

**Respuesta:** A) Process Control Block

---

## 14. ¿Cuál de los siguientes datos forma parte del PCB?
A) Estado del proceso  
B) Contador del programa  
C) Registros de CPU  
D) Todas las anteriores  

**Respuesta:** D) Todas las anteriores

---

## 15. ¿Qué indica el estado de un proceso?
A) El color de la ventana  
B) En qué situación se encuentra el proceso  
C) El tamaño del monitor  
D) El tipo de teclado conectado  

**Respuesta:** B) En qué situación se encuentra el proceso

---

## 16. ¿Cuál es el orden correcto de algunos estados principales de un proceso?
A) Terminado → Nuevo → Listo  
B) Nuevo → Listo → En ejecución  
C) En espera → Nuevo → Terminado  
D) Listo → Terminado → Nuevo  

**Respuesta:** B) Nuevo → Listo → En ejecución

---

## 17. ¿Qué significa que un proceso esté en estado "listo"?
A) Ya terminó  
B) Está esperando un evento de E/S  
C) Está esperando ser asignado a la CPU  
D) Está apagado  

**Respuesta:** C) Está esperando ser asignado a la CPU

---

## 18. ¿Qué significa que un proceso esté "en espera"?
A) Está ejecutándose normalmente  
B) Está esperando que ocurra un evento, como una operación de E/S  
C) Ya fue eliminado  
D) Nunca se ejecutará  

**Respuesta:** B) Está esperando que ocurra un evento, como una operación de E/S

---

## 19. ¿Qué ocurre cuando un proceso entra en estado "terminado"?
A) Gana prioridad  
B) Se vuelve proceso padre  
C) Se liberan sus recursos  
D) Se convierte en BIOS  

**Respuesta:** C) Se liberan sus recursos

---

## 20. ¿Qué es un proceso en foreground?
A) Un proceso interactivo en primer plano  
B) Un proceso que nunca usa CPU  
C) Un proceso apagado  
D) Un proceso del BIOS  

**Respuesta:** A) Un proceso interactivo en primer plano
